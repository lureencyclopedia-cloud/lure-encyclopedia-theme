# Lure Encyclopedia Design Principles

Version 1.0

---

# Purpose

Lure Encyclopedia exists to document the history, evolution, and practical use of fishing lures.

The website should feel like a respected editorial publication—not a blog, ecommerce store, or social media brand.

Every design decision should reinforce trust, readability, and long-term usefulness.

---

# Core Principles

## Editorial First

Content is always more important than decoration.

Typography, spacing, and photography should lead the experience.

---

## Timeless Over Trendy

Avoid design trends that will feel dated.

Do not use:

- Large gradients
- Glassmorphism
- Excessive animations
- Oversized shadows
- Rounded "app" interfaces

Prefer:

- Clean typography
- Thin borders
- White space
- Strong photography
- Editorial layouts

---

## Photography Leads

Photography is the primary visual element.

Images should appear museum-quality:

- Accurate colors
- Consistent lighting
- Large presentation
- Minimal cropping

Images should never compete with decorative graphics.

---

## Components Before Pages

Build reusable components first.

Examples:

- Navigation
- Hero
- Editorial Card
- Archive List
- Image Caption
- Pull Quote
- Citation
- Timeline Entry

Pages should be assembled from components.

---

## Typography

Typography should resemble a printed reference book.

Hierarchy should come from:

- Size
- Weight
- Spacing

—not excessive colors.

---

## Accessibility

Every component should:

- Support keyboard navigation
- Meet WCAG contrast guidelines
- Use semantic HTML
- Include alt text where appropriate

Accessibility is a requirement, not an enhancement.

---

## Performance

Every feature should justify its cost.

Avoid unnecessary JavaScript.

Prefer CSS whenever possible.

Fast pages improve usability and search visibility.

---

## Motion

Motion should be subtle.

Use transitions only to communicate interaction.

Avoid decorative animation.

---

## Consistency

Every new feature should match the existing design language.

Avoid one-off components whenever possible.

If a new pattern is needed, add it to the component library first.

---

# AI Rules

When generating code:

- Follow these principles before making design decisions.
- Prefer simple solutions over clever ones.
- Never redesign existing components without instruction.
- Stop after completing the requested milestone.
- Summarize all changes before requesting approval.---
